

### **PR-level metrics**

1. **Code longevity** – how long the code remained unchanged before being modified
2. **Number of dedicated bug-fix commits** related to this PR
3. **Lead time to merge** – time from when the developer started working on the feature until it was merged
4. **PR iterations** – number of revisions and number of review comments

---

### **Organization-level metrics**

1. **Per sprint**

   * Model usage distribution (percentage per model)
   * Delivery time
   * Model rating / performance score

2. **Per project**

   * Model usage distribution (percentage per model)
   * Model rating / performance score

3. **Per quarter**

   * Model usage distribution (percentage per model)
   * Model rating / performance score

---

### **Comparisons & insights**

* Compare **per developer / per team** to identify which model provides the best outcomes for them (not just global model comparison)
* Compare models **within the same system/domain only** (avoid cross-domain bias)
* Segment comparisons by **seniority level**


##


**Model Performance Score** is currently calculated as:

```
score = 0.35*LongevityScore + 0.30*BugFixScore + 0.20*LeadTimeScore + 0.15*IterationsScore
```

Each component is normalized to a 0–100 range based on the currently loaded dataset:

**LongevityScore:**
Higher = better.
Based on `longevity_days` (time until the first override; if there is no override, then up to a 60-day horizon after the last commit in the data).

**BugFixScore:**
Lower = better.
Based on `bugFixOverridesCount`.

**LeadTimeScore:**
Lower = better.
Based on `mergeDate - commitDate` in hours.

**IterationsScore:**
Lower = better.
Based on:

```
iterations_raw = revisionsBeforeMerge + commentsBeforeMerge / 4
```

---

Additionally, the following is calculated:

```
cost_performance_point = estimated_cost / max(score, 1.0)
```

Token estimate:

```
estimated_tokens = lines * 15
```

Cost (`estimated_cost`) follows subscription-aware logic:

1) If the developer used a model they are subscribed to:

```
estimated_cost = subscription_cost / number_of_commits_by_that_author_on_that_model
```

2) If they used a model they are not subscribed to:

```
estimated_cost = (estimated_tokens / 1e6) * cost_per_million
```

`cost_per_million` and `subscription_cost` are taken from `pricing.json`.
